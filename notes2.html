React Hooks: useMemo aur useCallback

📌 useMemo kya hota hai?

useMemo ek hook hai jo kisi value (jaise filtered list, calculation ka result) ko yaad (memoize) karta hai.

💡 Use Case:

  • Jab apko koi mehngi calculation har render par repeat nahi karni.
  • Jab ap filtered data bana rahe ho (e.g., search).
  • Performance ko better banane ke liye.

✅ Kyun use karein?

Bar bar same calculation se bacha ja sakta hai, sirf tab calculate kare jab dependencies (like input value) change ho.

📌 useCallback kya hota hai?

useCallback ek hook hai jo ek function ko yaad rakhta hai — taake har render par naye function ka reference create na ho.

💡 Use Case:

  • Agar aap ek function ko child component ko bhej rahe ho.
  • React.memo ke sath jab unnecessary re-render se bachna ho.
  • Jab function har baar same kaam karta hai, lekin baar baar naya ban raha ho.

✅ Kyun use karein?

Child component unnecessary render na kare, jab function ki value waise hi ho.

🎯 Summary:

  • useMemo => value yaad rakhta hai (e.g., filtered list)
  • useCallback => function yaad rakhta hai (e.g., onClick handler)
  • Dono ka use performance optimization ke liye hota hai

📚 React Context API

🔍 Context API kya hota hai?

Context API ek system hai jisse hum data ko ek component se dusre component tak bhej sakte hain bina props ke .

💡 Use Case:

  • Jab aapko deeply nested components tak data bhejna ho.
  • Jab aapko bar bar props drilling se bachna ho (matlab data ko Parent → Child → GrandChild sab jagah props se bhejna).
  • Example: theme, language, login user data, cart items, counter value, etc.

✅ Steps to Use Context API:

  1. 1. Create context using createContext()
  2. 2. Wrap parent component with Context.Provider
  3. 3. Use useContext() in child to access data

🎯 Fayda kya hai?

  • Code clean ho jata hai
  • Props drilling ki problem khatam ho jati hai
  • Components re-use karna easy ho jata hai

📘 React useRef Hook

🔍 useRef kya hota hai?

useRef ek hook hai jo hume kisi bhi DOM element ka reference deta hai bina component ko re-render kiye.

💡 Kab use karte hain?

  • Input field ko automatically focus dena ho
  • Koi bhi DOM element access karna ho (jaise button, div, etc)
  • Ek value ko store karna ho jo re-render pe change na ho
  • Timers, previous values, etc handle karne ke liye

✅ Example:

const inputRef = useRef(null);
<input ref={inputRef} />
inputRef.current.focus();

🎯 Fayda kya hai?

  • DOM directly control kar sakte hain
  • Kisi bhi cheez ka reference rakha ja sakta hai bina render ke
  • Performance improve hoti hai (no extra re-renders)
